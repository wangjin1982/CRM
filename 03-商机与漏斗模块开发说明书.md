# 模块三：商机与销售漏斗模块开发说明书

## 📋 模块概述

### 模块名称
商机与销售漏斗管理模块（Opportunity & Sales Funnel Management）

### 模块定位
CRM系统的核心业务模块，负责商机的全生命周期管理，包括商机创建、阶段推进、销售漏斗可视化等。

### 开发优先级
**P1（高优先级）** - 依赖客户模块，支撑销售行为管理

---

## 🎯 模块目标

### 主要目标
1. 实现完整的商机CRUD管理
2. 实现销售阶段定义与流转
3. 实现销售漏斗可视化
4. 实现商机与客户、联系人的多对多关联
5. 实现商机预测与统计

### 业务价值
- 帮助销售团队管理销售机会
- 可视化销售漏斗，识别转化瓶颈
- 支持销售预测和业绩分析

## 📐 数据库设计

### 表结构设计

#### 1. 商机信息表 (crm_opportunity_info)
```sql
CREATE TABLE crm_opportunity_info (
    id                  BIGSERIAL PRIMARY KEY,
    opportunity_no      VARCHAR(50) NOT NULL UNIQUE,   -- 商机编号
    opportunity_name    VARCHAR(200) NOT NULL,         -- 商机名称

    -- 关联信息
    customer_id         BIGINT NOT NULL REFERENCES crm_customer_info(id),
    customer_name       VARCHAR(200),                  -- 客户名称（冗余）
    primary_contact_id  BIGINT REFERENCES crm_contact_info(id), -- 主要联系人

    -- 金额信息
    estimated_amount    DECIMAL(20,2),                 -- 预估金额
    actual_amount       DECIMAL(20,2),                 -- 实际成交金额
    currency            VARCHAR(10) DEFAULT 'CNY',     -- 币种

    -- 阶段信息
    stage_id            BIGINT NOT NULL REFERENCES crm_stage_def(id), -- 当前阶段
    stage_name          VARCHAR(50),                   -- 阶段名称（冗余）
    stage_order         INT,                           -- 阶段序号

    -- 时间信息
    expected_close_date DATE,                          -- 预计成交日期
    actual_close_date   DATE,                          -- 实际成交日期

    -- 概率与优先级
    win_probability     INT DEFAULT 50,                -- 成交概率(%)
    priority            VARCHAR(20) DEFAULT 'medium',  -- 优先级: high/medium/low

    -- 负责人信息
    owner_id            BIGINT REFERENCES sys_user(id),-- 负责人
    owner_name          VARCHAR(50),

    -- 竞争信息
    competitors         TEXT,                          -- 竞争对手
    competitive_status VARCHAR(50),                    -- 竞争状态

    -- 产品信息
    products            JSONB,                         -- 产品信息
    -- [
    --   {"name": "产品A", "quantity": 10, "price": 1000, "amount": 10000}
    -- ]

    -- 来源信息
    lead_source         VARCHAR(50),                   -- 商机来源

    -- 状态
    status              VARCHAR(20) DEFAULT 'open',    -- open/won/lost/abandoned

    -- AI分析字段
    risk_level          VARCHAR(20),                   -- 风险等级: low/medium/high
    risk_factors        JSONB,                         -- 风险因素
    ai_suggestions      TEXT,                          -- AI建议
    last_ai_analysis    TIMESTAMP,                     -- 最后AI分析时间

    -- 停滞预警
    days_in_stage       INT DEFAULT 0,                 -- 当前阶段停留天数
    is_stagnant         BOOLEAN DEFAULT FALSE,         -- 是否停滞
    stagnant_alert_sent BOOLEAN DEFAULT FALSE,         -- 是否已发送预警

    -- 统计信息
    activity_count      INT DEFAULT 0,                 -- 活动次数
    last_activity_at    TIMESTAMP,                     -- 最后活动时间

    -- 元数据
    description         TEXT,                          -- 商机描述
    lost_reason         VARCHAR(200),                  -- 流失原因
    tags                VARCHAR(500)[],
    created_by          BIGINT REFERENCES sys_user(id),
    updated_by          BIGINT REFERENCES sys_user(id),
    created_at          TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at          TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    deleted_at          TIMESTAMP
);

-- 索引
CREATE INDEX idx_crm_opportunity_customer ON crm_opportunity_info(customer_id);
CREATE INDEX idx_crm_opportunity_owner ON crm_opportunity_info(owner_id);
CREATE INDEX idx_crm_opportunity_stage ON crm_opportunity_info(stage_id);
CREATE INDEX idx_crm_opportunity_status ON crm_opportunity_info(status);
CREATE INDEX idx_crm_opportunity_close_date ON crm_opportunity_info(expected_close_date);
CREATE INDEX idx_crm_opportunity_stagnant ON crm_opportunity_info(is_stagnant);
```

#### 2. 销售阶段定义表 (crm_stage_def)
```sql
CREATE TABLE crm_stage_def (
    id              BIGSERIAL PRIMARY KEY,
    stage_name      VARCHAR(50) NOT NULL,
    stage_code      VARCHAR(50) NOT NULL UNIQUE,
    stage_order     INT NOT NULL,
    stage_type      VARCHAR(20) DEFAULT 'normal', -- normal, won, lost
    probability     INT DEFAULT 0,                -- 默认成交概率
    duration_days   INT,                          -- 建议停留天数
    is_active       BOOLEAN DEFAULT TRUE,
    description     VARCHAR(200),
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 初始数据
INSERT INTO crm_stage_def (stage_name, stage_code, stage_order, probability, duration_days) VALUES
('线索', 'lead', 1, 10, 7),
('需求确认', 'qualified', 2, 20, 14),
('方案报价', 'proposal', 3, 40, 21),
('谈判协商', 'negotiation', 4, 60, 30),
('合同审核', 'contract', 5, 80, 14),
('赢单', 'won', 6, 100, NULL),
('输单', 'lost', 7, 0, NULL);
```

#### 3. 商机联系人关联表 (crm_opportunity_contact)
```sql
CREATE TABLE crm_opportunity_contact (
    id              BIGSERIAL PRIMARY KEY,
    opportunity_id  BIGINT NOT NULL REFERENCES crm_opportunity_info(id),
    contact_id      BIGINT NOT NULL REFERENCES crm_contact_info(id),
    role            VARCHAR(50),     -- 在此商机中的角色
    influence_level SMALLINT,        -- 影响力等级
    is_primary      BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(opportunity_id, contact_id)
);

CREATE INDEX idx_opp_contact_opp ON crm_opportunity_contact(opportunity_id);
CREATE INDEX idx_opp_contact_contact ON crm_opportunity_contact(contact_id);
```

#### 4. 阶段变更历史表 (crm_opportunity_stage_history)
```sql
CREATE TABLE crm_opportunity_stage_history (
    id              BIGSERIAL PRIMARY KEY,
    opportunity_id  BIGINT NOT NULL REFERENCES crm_opportunity_info(id),
    from_stage_id   BIGINT REFERENCES crm_stage_def(id),
    to_stage_id     BIGINT NOT NULL REFERENCES crm_stage_def(id),
    stage_duration  INT,             -- 在上一阶段停留天数
    changed_by      BIGINT REFERENCES sys_user(id),
    changed_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    notes           TEXT             -- 变更说明
);

CREATE INDEX idx_opp_stage_history_opp ON crm_opportunity_stage_history(opportunity_id);
```

#### 5. 竞争对手信息表 (crm_competitor)
```sql
CREATE TABLE crm_competitor (
    id              BIGSERIAL PRIMARY KEY,
    opportunity_id  BIGINT NOT NULL REFERENCES crm_opportunity_info(id),
    competitor_name VARCHAR(100) NOT NULL,
    strength        TEXT,            -- 优势
    weakness        TEXT,            -- 劣势
    price_offer     DECIMAL(20,2),   -- 对手报价
    threat_level    SMALLINT,        -- 威胁等级 1-5
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
