# 模块六：数据分析与可视化模块开发说明书

## 📋 模块概述

### 模块名称
数据分析与可视化模块（Data Analytics & Visualization）

### 模块定位
负责系统数据的统计、分析和可视化展示，为管理层和销售人员提供数据决策支持。

### 开发优先级
**P1（高优先级）** - 依赖所有业务模块，提供数据洞察

---

## 🎯 模块目标

### 主要目标
1. 实现多维度数据统计
2. 实现可视化仪表盘
3. 实现自定义报表
4. 实现数据导出
5. 提供移动端适配的视图

### 业务价值
- 实时掌握业务动态
- 发现业务问题和机会
- 支持数据驱动决策
- 提升管理效率

## 📐 数据库设计

### 表结构设计

#### 1. 报表定义表 (crm_report_def)
```sql
CREATE TABLE crm_report_def (
    id              BIGSERIAL PRIMARY KEY,
    report_name     VARCHAR(100) NOT NULL,
    report_code     VARCHAR(50) NOT NULL UNIQUE,
    report_type     VARCHAR(50),     -- dashboard, chart, table
    category        VARCHAR(50),     -- customer, opportunity, activity, performance

    -- 数据配置
    data_source     TEXT,           -- 数据源配置（JSON）
    query_config    JSONB,          -- 查询配置
    chart_config    JSONB,          -- 图表配置

    -- 显示配置
    is_public       BOOLEAN DEFAULT FALSE,
    allowed_roles   BIGINT[],       -- 可访问的角色

    -- 定时配置
    is_scheduled    BOOLEAN DEFAULT FALSE,
    schedule_type   VARCHAR(20),    -- daily, weekly, monthly
    schedule_time   TIME,

    -- 元数据
    description     TEXT,
    created_by      BIGINT REFERENCES sys_user(id),
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. 报表订阅表 (crm_report_subscription)
```sql
CREATE TABLE crm_report_subscription (
    id              BIGSERIAL PRIMARY KEY,
    report_id       BIGINT NOT NULL REFERENCES crm_report_def(id),
    user_id         BIGINT NOT NULL REFERENCES sys_user(id),
    delivery_method VARCHAR(50),     -- email, in_app, both
    delivery_time   TIME,

    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(report_id, user_id)
);
```

#### 3. 数据快照表 (crm_data_snapshot)
```sql
CREATE TABLE crm_data_snapshot (
    id              BIGSERIAL PRIMARY KEY,
    snapshot_type   VARCHAR(50) NOT NULL,
    snapshot_date   DATE NOT NULL,
    snapshot_data   JSONB NOT NULL,

    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(snapshot_type, snapshot_date)
);

-- 用于存储每日统计数据快照
```

#### 4. KPI定义表 (crm_kpi_def)
```sql
CREATE TABLE crm_kpi_def (
    id              BIGSERIAL PRIMARY KEY,
    kpi_name        VARCHAR(100) NOT NULL,
    kpi_code        VARCHAR(50) NOT NULL UNIQUE,
    kpi_category    VARCHAR(50),     -- customer, opportunity, performance

    -- KPI配置
    calculation_formula TEXT,        -- 计算公式
    target_value    DECIMAL(20,2),   -- 目标值
    threshold_warning DECIMAL(20,2), -- 警告阈值
    threshold_danger DECIMAL(20,2),  -- 危险阈值

    -- 显示配置
    unit            VARCHAR(20),     -- 单位
    display_type    VARCHAR(50),     -- number, percentage, currency
    chart_type      VARCHAR(50),     -- line, bar, gauge, etc
    trend_days      INT DEFAULT 30,  -- 趋势显示天数

    -- 权限
    is_public       BOOLEAN DEFAULT FALSE,
    allowed_roles   BIGINT[],

    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 初始化KPI
INSERT INTO crm_kpi_def (kpi_name, kpi_code, kpi_category, calculation_formula, target_value, unit, display_type, chart_type) VALUES
('新增客户数', 'new_customers', 'customer', 'COUNT(DISTINCT customer_id WHERE created_at >= today)', 50, '个', 'number', 'bar'),
('商机总数', 'total_opportunities', 'opportunity', 'COUNT(*) WHERE status = ''open''', 100, '个', 'number', 'line'),
('赢单金额', 'won_amount', 'opportunity', 'SUM(actual_amount) WHERE status = ''won'' AND close_date >= this_month', 500000, '元', 'currency', 'bar'),
('销售漏斗转化率', 'funnel_conversion', 'performance', '(won_count / lead_count) * 100', 20, '%', 'percentage', 'gauge');
```
