# CRM系统 V1 销售SOP与版本说明书

- 版本：V1.0.0
- 日期：2026-02-27
- 项目：`wangjin1982/CRM`

## 1. 文档目标

本说明书包含三部分：

1. 以销售总监视角定义 CRM 日常使用 SOP（线索到回款的执行流程）。
2. 以开发总监视角对照 SOP 评估系统现状，并给出 V1 优化结论。
3. 输出 V1 模块与功能清单，明确“当前可用范围”和“后续迭代方向”。

## 2. 角色定义

- `TO`：Technical Owner（技术支持人员）
- `TBM`：Technical Business Manager（技术经理）
- `RTM`：Regional Technical Manager（区域技术经理）
- `ISM`：区域销售经理
- `销售`：客户经营、商机推进、活动执行的第一责任人
- `销售总监`：节奏管理、资源协调、预测与复盘负责人

## 3. 销售总监视角 SOP（系统使用）

### 3.1 日启动（每天）

1. 进入控制台查看 4 个核心指标：客户总数、Open商机数、Open金额（万元）、待办任务。
2. 打开销售漏斗，检查各阶段数量、转化率、停滞率。
3. 重点查看停滞商机（阶段停留过久）并确认当日推进动作。
4. 按负责人维度检查任务超期情况，安排 ISM 跟进。

### 3.2 客户经营（每周）

1. 在客户管理中筛选高价值客户（3#客户分级、5#成交客户、负责人）。
2. 抽查客户详情：联系人完整性、交互记录完整性、客户名下商机完整性。
3. 对空缺关键字段（官网/地址/公司信息/产品信息）使用 AI 工作台进行补全建议。
4. 对客户进行省份/城市维度复盘，识别区域资源覆盖缺口。

### 3.3 商机推进（每周）

1. 在商机列表按阶段、优先级、状态筛选，识别 Top Pipeline。
2. 进入商机详情，按顺序推进阶段（Lead -> ... -> Closed Won/Closed Lost）。
3. 必查项：
   - 阶段定义透传（当前阶段标准是否满足）
   - 活动记录（拜访/跟进/任务/日程是否支持当前阶段）
   - 下一步建议（系统规则建议是否已执行）
4. 对预计成交日期临近但概率偏低商机，组织 TO/TBM 联合评审。

### 3.4 行为管理（每周）

1. 活动看板查看拜访、跟进、任务、日程总量与节奏。
2. 对关键客户要求每周至少有一条有效交互记录。
3. 对关键商机要求至少有“任务 + 跟进”的闭环动作证据。

### 3.5 月度复盘（每月）

1. 分析看板输出：新增客户、阶段转化、赢单/输单、行为投入产出。
2. 复盘 Closed Lost：主要原因、竞争对手、可改进动作。
3. 更新阶段策略：哪些阶段停滞严重，是否需要调整 TO/TBM 介入时点。
4. 输出下月行动清单并转成任务（日程 + 负责人 + 截止日期）。

## 4. 开发总监评估与 V1 优化结论

### 4.1 对照 SOP 的核心判断

1. 客户、商机、活动三大对象已打通，具备基础 CRM 闭环。
2. 导入链路已可用，支持从外部 CRM 报表迁移历史数据。
3. AI 与分析能力已有首版，但仍以规则引擎和运营分析为主。
4. 部分页面存在“流程可走但深度不足”的问题，需要补强操作细节。

### 4.2 本次 V1 已完成优化

1. 客户详情补齐“交互记录”可视化与新增能力（不再是占位页）。
2. 客户详情补齐“活动记录”联动（拜访/跟进/任务/日程）。
3. 商机概览新增“下一步建议”规则卡（停滞、活动不足、close窗口风险、阶段标准提醒）。
4. 负责人与区域数据结构优化：
   - 负责人支持销售人员下拉/搜索选择；
   - 区域拆分为省份+城市并统一展示。
5. 控制台金额展示统一为“万元”口径（便于管理层审阅）。

### 4.3 V1 不做项（明确边界）

1. 暂不引入复杂审批流与多层组织权限。
2. 暂不引入在线文档协同与报价审批引擎。
3. 暂不引入训练型机器学习模型（先使用规则与统计分析）。

## 5. V1 模块与功能清单

### 模块1：基础设施

- 用户认证登录、角色基础框架
- FastAPI + Vue3 + SQLite/PostgreSQL 双模式
- 一键脚本：初始化、启动、冒烟测试

### 模块2：客户资源管理

- 客户列表/详情/编辑
- 字段体系：1#客户名称（中/EN）、区域（省市）、客户类型/分级、成交标识、电气工程师人数、负责人
- 联系人管理、客户360视图
- 客户交互记录（新增/查看/删除）
- Excel 导入导出

### 模块3：商机与漏斗

- 商机新增、编辑、详情、删除
- 销售阶段体系（含 Lead、Closed Won、Closed Lost）
- 阶段顺序推进、阶段历史
- 商机详情联动活动记录
- 漏斗分析（阶段分布、转化、停滞、建议）

### 模块4：销售行为管理

- 拜访、跟进、任务、日程 CRUD
- 客户维度与商机维度过滤
- 活动统计看板

### 模块5：AI 智能增强

- API 配置（可配置 GLM Key）
- 客户属性补全建议
- 商机风险分析与预警
- 拜访总结、漏斗智能建议、自然语言查询（规则版）

### 模块6：数据分析

- 首页看板、销售仪表盘、管理驾驶舱
- 客户/商机/活动统计
- 报表定义、执行、CSV 导出

### 模块7：测试与集成

- 后端冒烟测试通过（当前 5 项）
- 前端构建可通过
- 核心流程联调：客户 -> 商机 -> 活动 -> AI/分析

## 6. 数据与导入策略（V1）

1. 主数据：客户、商机、活动记录。
2. 导入方式：保持你当前 Excel 模板结构不变，按批次增量导入。
3. 去重原则：
   - 客户：客户名称
   - 商机：客户 + 商机名称
   - 活动：外部活动ID/编号
4. 建议：每次导入保留日志（导入时间、成功数、失败数、失败原因）。

## 7. 数据库建议（Mac 场景）

1. 本地开发优先使用 `SQLite`（安装成本最低，开箱即用）。
2. 团队协作/生产环境建议使用 `PostgreSQL`。
3. 当前不建议使用 `SQL Server`（Mac 本地开发维护成本高，生态不如前两者直接）。

## 8. V1 验收标准

1. 能登录并进入控制台，指标可展示。
2. 客户列表、客户详情、商机列表、商机详情均有真实数据。
3. 客户详情可看到交互记录与活动记录。
4. 商机详情可推进阶段并看到“下一步建议”。
5. 导入脚本可重复执行，且不会产生大量重复数据。

## 9. V1 后续迭代（V1.1建议）

1. 商机赢率模型（基于阶段流转、活动密度、停留时长）。
2. 客户流失预警模型（跟进间隔、活跃度、历史成单）。
3. 导入任务日志页面（可视化每次导入结果）。
4. 角色权限深化（TO/TBM/RTM/ISM 的数据可见范围与审批权限）。
