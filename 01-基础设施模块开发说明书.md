# 模块一：基础设施模块开发说明书

## 📋 模块概述

### 模块名称
基础设施与核心服务模块（Infrastructure & Core Services）

### 模块定位
项目的基础支撑模块，为所有其他业务模块提供统一的技术基础，包括数据库设计、API规范、认证鉴权、日志监控等核心能力。

### 开发优先级
**P0（最高优先级）** - 需最先完成，其他模块依赖此模块

---

## 🎯 模块目标

### 主要目标
1. 搭建项目基础框架和开发环境
2. 设计并实现核心数据模型
3. 建立统一的API规范和接口标准
4. 实现完整的认证鉴权体系
5. 搭建日志、监控和错误处理机制

### 非目标
- 具体业务功能实现（由其他模块负责）
- 复杂的业务逻辑（只提供基础支持）

### 核心表结构

#### 1. 用户表 (sys_user)
```sql
CREATE TABLE sys_user (
    id              BIGSERIAL PRIMARY KEY,
    username        VARCHAR(50) NOT NULL UNIQUE,
    email           VARCHAR(100) NOT NULL UNIQUE,
    phone           VARCHAR(20),
    password_hash   VARCHAR(255) NOT NULL,
    real_name       VARCHAR(50),
    avatar          VARCHAR(255),
    status          SMALLINT DEFAULT 1, -- 1:启用 0:禁用
    department_id   BIGINT,
    position        VARCHAR(50),
    is_admin        BOOLEAN DEFAULT FALSE,
    last_login_at   TIMESTAMP,
    last_login_ip   VARCHAR(50),
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    deleted_at      TIMESTAMP
);

CREATE INDEX idx_sys_user_email ON sys_user(email);
CREATE INDEX idx_sys_user_status ON sys_user(status);
```

#### 2. 角色表 (sys_role)
```sql
CREATE TABLE sys_role (
    id          BIGSERIAL PRIMARY KEY,
    name        VARCHAR(50) NOT NULL UNIQUE,
    code        VARCHAR(50) NOT NULL UNIQUE,
    description TEXT,
    status      SMALLINT DEFAULT 1,
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3. 用户角色关联表 (sys_user_role)
```sql
CREATE TABLE sys_user_role (
    id         BIGSERIAL PRIMARY KEY,
    user_id    BIGINT NOT NULL REFERENCES sys_user(id),
    role_id    BIGINT NOT NULL REFERENCES sys_role(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, role_id)
);
```

#### 4. 权限表 (sys_permission)
```sql
CREATE TABLE sys_permission (
    id          BIGSERIAL PRIMARY KEY,
    parent_id   BIGINT DEFAULT 0,
    name        VARCHAR(50) NOT NULL,
    code        VARCHAR(100) NOT NULL UNIQUE,
    type        VARCHAR(20) NOT NULL, -- menu, button, api
    path        VARCHAR(200),
    method      VARCHAR(10), -- GET, POST, PUT, DELETE
    icon        VARCHAR(50),
    sort_order  INT DEFAULT 0,
    status      SMALLINT DEFAULT 1,
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 5. 角色权限关联表 (sys_role_permission)
```sql
CREATE TABLE sys_role_permission (
    id            BIGSERIAL PRIMARY KEY,
    role_id       BIGINT NOT NULL REFERENCES sys_role(id),
    permission_id BIGINT NOT NULL REFERENCES sys_permission(id),
    created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(role_id, permission_id)
);
```

#### 6. 系统配置表 (sys_config)
```sql
CREATE TABLE sys_config (
    id          BIGSERIAL PRIMARY KEY,
    config_key  VARCHAR(100) NOT NULL UNIQUE,
    config_value TEXT,
    config_type VARCHAR(20), -- string, number, boolean, json
    description VARCHAR(200),
    is_public   BOOLEAN DEFAULT FALSE, -- 是否可向前端暴露
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 7. 操作日志表 (sys_operation_log)
```sql
CREATE TABLE sys_operation_log (
    id          BIGSERIAL PRIMARY KEY,
    user_id     BIGINT REFERENCES sys_user(id),
    username    VARCHAR(50),
    module      VARCHAR(50),
    operation   VARCHAR(50),
    method      VARCHAR(10),
    path        VARCHAR(200),
    params      TEXT,
    ip          VARCHAR(50),
    user_agent  TEXT,
    status      SMALLINT, -- 1:成功 0:失败
    error_msg   TEXT,
    duration    INT, -- 请求耗时(ms)
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_sys_operation_log_user ON sys_operation_log(user_id);
CREATE INDEX idx_sys_operation_log_created ON sys_operation_log(created_at);
```

#### 8. 字典表 (sys_dict)
```sql
CREATE TABLE sys_dict (
    id          BIGSERIAL PRIMARY KEY,
    dict_type   VARCHAR(50) NOT NULL,
    dict_label  VARCHAR(100) NOT NULL,
    dict_value  VARCHAR(100) NOT NULL,
    sort_order  INT DEFAULT 0,
    status      SMALLINT DEFAULT 1,
    remark      VARCHAR(200),
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(dict_type, dict_value)
);

CREATE INDEX idx_sys_dict_type ON sys_dict(dict_type);
```

## 🔌 API设计规范

### RESTful API规范

#### 1. URL设计规范
```
# 资源型API
GET    /api/v1/users           # 获取列表
GET    /api/v1/users/{id}      # 获取详情
POST   /api/v1/users           # 创建
PUT    /api/v1/users/{id}      # 全量更新
PATCH  /api/v1/users/{id}      # 部分更新
DELETE /api/v1/users/{id}      # 删除

# 非资源型API
POST   /api/v1/users/{id}/activate     # 激活
POST   /api/v1/users/{id}/deactivate   # 停用
```

#### 2. 统一响应格式
```json
// 成功响应
{
    "code": 200,
    "message": "success",
    "data": {...},
    "timestamp": 1708723200000
}

// 列表响应
{
    "code": 200,
    "message": "success",
    "data": {
        "items": [...],
        "total": 100,
        "page": 1,
        "pageSize": 20
    },
    "timestamp": 1708723200000
}

// 错误响应
{
    "code": 400,
    "message": "参数验证失败",
    "errors": [
        {"field": "email", "message": "邮箱格式不正确"}
    ],
    "timestamp": 1708723200000
}
```

#### 3. HTTP状态码规范
| 状态码 | 含义 | 使用场景 |
|-------|------|---------|
| 200 | OK | 请求成功 |
| 201 | Created | 创建成功 |
| 400 | Bad Request | 参数错误 |
| 401 | Unauthorized | 未认证 |
| 403 | Forbidden | 无权限 |
| 404 | Not Found | 资源不存在 |
| 422 | Unprocessable Entity | 业务验证失败 |
| 500 | Internal Server Error | 服务器错误 |

#### 4. 业务错误码规范
```
# 通用错误码（1000-1999）
1001: 参数验证失败
1002: 数据不存在
1003: 数据已存在
1004: 操作频繁

# 认证授权错误码（2000-2999）
2001: 未登录
2002: 登录过期
2003: 无权限
2004: Token无效

# 业务错误码（5000-8999）
5000-5999: 客户模块
6000-6999: 商机模块
7000-7999: 行为模块
8000-8999: AI模块
```

## 🔐 认证鉴权设计

### 1. JWT认证方案
```
# Access Token（短期有效）
{
    "user_id": 1,
    "username": "admin",
    "exp": 1708723200,  # 2小时后过期
    "type": "access"
}

# Refresh Token（长期有效）
{
    "user_id": 1,
    "exp": 1708982400,  # 7天后过期
    "type": "refresh"
}
```

### 2. 权限控制模型
```
RBAC（基于角色的访问控制）

用户(User) ──多对多── 角色(Role) ──多对多── 权限(Permission)

权限粒度：
- 模块级：customer:read, opportunity:write
- 操作级：customer:view, customer:create, customer:update, customer:delete
- 数据级：own（自己的）, department（部门）, all（全部）
```

### 3. 认证流程
```
1. 登录
   POST /api/v1/auth/login
   → 返回 access_token + refresh_token

2. 访问API
   Header: Authorization: Bearer {access_token}

3. Token刷新
   POST /api/v1/auth/refresh
   Body: { "refresh_token": "..." }
   → 返回新的 access_token

4. 登出
   POST /api/v1/auth/logout
   → 将token加入黑名单
```
